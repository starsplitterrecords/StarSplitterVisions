# Content Field Ownership Map

Field ownership is split across source JSON authors, admin helper generation logic, and runtime validators/renderers.

| Field | Content type | Owned by | Generated by admin? | Used by public UI? | Required? | Notes |
|---|---|---|---|---|---|---|
| `series.slug` | Series | Content JSON author | No | Yes | Yes | Primary key; validator drops row if missing. |
| `series.title` | Series | Content JSON author | No | Yes | Yes (with fallback) | Missing title warns; fallback `Untitled Series`. |
| `series.heroImage` | Series | Content JSON author | No | Yes | Optional | Must be `/images/...` when present. |
| `series.thumbnailImage` | Series | Content JSON author | No | Yes | Optional | Used for cards/thumbnails when present. |
| `series.accentColor` | Series | Content JSON author | No | Yes | Optional | Visual theming helper. |
| `release.id` | Release | Content JSON author | Yes (`slug/id` input) | Yes | Yes (`id` or `slug`) | Runtime normalizes to `id = id || slug`. |
| `release.seriesSlug` | Release | Content JSON author | Yes | Yes | Yes | Must match `series.slug` logically. |
| `release.issueNumber` | Release | Content JSON author | Yes | Yes | Optional | Coerced to number by generator when possible. |
| `release.releaseDate` | Release | Content JSON author | Yes | Yes | Conditional | Required if `status=scheduled`. |
| `release.status` | Release | Content JSON author | Yes | Yes | Optional | Allowed values: `draft`, `scheduled`, `published`. |
| `release.coverImage` | Release | Content JSON author | Yes (defaulted) | Yes | Optional | Generator defaults to `/images/{series}/{release}/cover.jpg`. |
| `release.heroImage` | Release | Content JSON author | Yes (defaulted) | Yes | Optional | Generator defaults to `/images/{series}/{release}/hero.jpg`. |
| `release.image` | Release | Content JSON author | Yes (defaulted) | Yes | Optional | Card/preview image; generator defaults to cover image path. |
| `release.ctaLabel` | Release | Content JSON author | Yes | Yes | Optional | Defaults to `Read Issue` in generator form. |
| `page.releaseSlug` | Page | Content JSON author | Yes | Yes | Yes | Required for reader grouping and route matching. |
| `page.pageNumber` | Page | Content JSON author | Yes | Yes | Yes | Must be numeric for ordering/reader navigation. |
| `page.image` | Page | Content JSON author | Yes | Yes | Yes | Reader image path; should be `/images/...`. |
| `page.seriesSlug` | Page | Content JSON author | Yes | Yes | Optional (validator) | Integrity should align with parent release series. |
| `soundtrack.seriesSlug` | Soundtrack | Content JSON author | Package workflow/manual | Yes | Optional (soundtrack validator) | Required in stricter reader soundtrack validation path. |
| `soundtrack.releaseSlug` | Soundtrack | Content JSON author | Package workflow/manual | Yes | Optional | Links soundtrack to a release when present. |
| `soundtrack.tracks` | Soundtrack | Content JSON author | Package workflow/manual | Yes | Optional | Track list consumed by soundtrack UI/player. |
| `soundtrack.title` | Soundtrack | Content JSON author | Package workflow/manual | Yes | Yes | Soundtrack validator requires non-empty title. |
| `readerSoundtrack.id` | Reader Soundtrack | Runtime/transformed soundtrack item | No | Yes | Yes | Reader validator drops rows without `id`. |
| `readerSoundtrack.seriesSlug` | Reader Soundtrack | Runtime/transformed soundtrack item | No | Yes | Yes | Reader validator drops rows without `seriesSlug`. |
| `readerSoundtrack.title` | Reader Soundtrack | Runtime/transformed soundtrack item | No | Yes | Optional (with fallback) | Fallback `Untitled Soundtrack`. |
| `extra.title` | Extra | Content JSON author | Package workflow/manual | Yes | Yes | Extras validator requires title. |
| `extra.seriesSlug` | Extra | Content JSON author | Package workflow/manual | Yes | Optional | Used for grouping/association. |
| `extra.url` | Extra | Content JSON author | Package workflow/manual | Yes | Optional | Public destination path/link. |
| `asset.repoPath` | Asset | Repository file layout | Yes (path guidance) | Indirect | N/A | Physical file location must be under `public/images/...`. |
| `asset.publicPath` | Asset | JSON field value | Yes (path guidance/defaults) | Yes | N/A | Must use `/images/...`, never `/public/images/...`. |
| `codexPackage.targetRepoPaths[]` | Admin-generated Codex package | Admin/package author | Yes | No | Required in workflow | Handoff metadata for files Codex should update. |
| `codexPackage.targetJsonPaths[]` | Admin-generated Codex package | Admin/package author | Yes | No | Required in workflow | Points to JSON files under `public/content/...`. |
| `codexPackage.releaseOrPageMetadata` | Admin-generated Codex package | Admin/package author | Yes | No | Required in workflow | Includes series/release/page metadata for patching. |
| `codexPackage.acceptanceCriteria[]` | Admin-generated Codex package | Admin/package author | Yes | No | Required in workflow | Captures validation/review checklist for PR readiness. |

## Ownership notes

- **Content JSON author** owns canonical persisted values in `public/content/*.json`.
- **Admin generator** provides defaults/scaffold output and warnings, but does not write files directly.
- **Runtime validators** enforce hard requirements at load time and may drop invalid rows or apply fallback display values.
